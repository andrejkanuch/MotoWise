# Types Package — @motolearn/types

## Purpose
Shared TypeScript types, Zod validation schemas, and constants for the MotoLearn monorepo.

## Structure
- `src/constants/enums.ts` — shared enum-like `as const` objects (UserRole, ArticleDifficulty, etc.)
- `src/constants/limits.ts` — business rule constants (free tier limits, file size limits)
- `src/validators/*.ts` — Zod schemas for input validation
- `src/database.types.ts` — auto-generated by `supabase gen types` (DO NOT edit manually)

## Rules
- Always export BOTH the Zod schema AND the inferred TypeScript type
- Use `as const` objects, never TypeScript `enum`
- QuizSchema lives in `validators/quiz.ts`, NOT in `article.ts`
- database.types.ts is auto-generated — run `pnpm db:types` to update
- This package is imported by all apps — keep it dependency-light (only zod)
