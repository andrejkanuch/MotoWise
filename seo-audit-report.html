MotoVault.app — SEO Audit

Full site audit • March 14, 2026 • Domain: motovault.app

Executive Summary

MotoVault has a solid technical SEO foundation — proper meta tags, structured data (Organization + FAQPage), i18n with hreflang, strong security headers, and a clean URL structure. The biggest strength is the well-implemented internationalization across 5 locales with proper alternate URLs. However, the site's top 3 priorities are: (1) the sitemap is critically incomplete — only 3 pages are indexed while 5 public pages exist, and the support page is entirely missing; (2) there is zero blog or content marketing, meaning the site cannot rank for informational keywords that drive organic traffic in this space; and (3) competitors like MotoDoc, FixioMoto, and Moto Shed are actively publishing SEO content while MotoVault relies solely on a single landing page. Overall assessment: strong technical foundation, but severely limited content footprint.

Technical SEO Checklist

  • HTTPS enabled with HSTS (max-age=63072000, includeSubDomains, preload)
  • robots.txt properly configured — allows / , disallows /admin/, includes sitemap URL
  • XML sitemap exists with localized alternates and hreflang tags
  • Proper viewport meta tag (width=device-width, initialScale=1)
  • Font loading optimized (display: 'swap' on Google Fonts)
  • DNS prefetch for Supabase API endpoint
  • Strong security headers (CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy)
  • Semantic HTML with skip-to-content link for accessibility
  • Language tag set dynamically on <html> element based on locale
  • Open Graph and Twitter Card tags configured globally
  • FAQPage and Organization structured data on homepage
  • Next.js React Compiler enabled for performance
  • Sitemap missing /support and /account-deletion pages
  • Sitemap missing fr and it locales (only includes en, es, de)
  • No explicit canonical URLs on marketing pages (relying on Next.js defaults)
  • No SoftwareApplication schema for app store discoverability
  • No breadcrumb structured data
  • CSP connect-src limited to 'self' — may block analytics if added later
  • No web analytics installed (no Google Analytics, Plausible, or similar)
  • No Google Search Console verification meta tag detected (file-based verification exists)
  • No blog/content section — severely limits crawlable and indexable pages
