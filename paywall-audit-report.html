Critical Pricing Mismatch

The paywall shows pricing that does not match the documented pricing in your PRD and project documentation.

Plan Paywall (RevenueCat) PRD / Docs Status
Monthly $9.99/mo $6.99/mo (3-day trial) Mismatch
Annual $79.99/yr ($6.67/mo) $39.99/yr (7-day trial) Mismatch
Discount Badge 19% OFF 52% savings Mismatch
Action Decide on final pricing
Either update RevenueCat products to $6.99/mo and $39.99/yr to match the PRD, or update all documentation/onboarding copy to reflect the new $9.99/$79.99 pricing. The discount badge ("19% OFF") is auto-calculated by RevenueCat's relative_discount variable — it will update automatically once real product prices are set in App Store Connect / Google Play Console.
Note Free trial info missing from paywall
Your PRD specifies 3-day trial for monthly and 7-day trial for annual. The RevenueCat paywall doesn't display trial information. The in-app upgrade screen (upgrade.tsx) shows trial days, but the RevenueCat remote paywall during onboarding does not. Consider adding trial messaging to improve conversion.

Critical Feature Carousel vs. Reality

The paywall carousel advertises 3 feature pages. Here's how they map to actual built functionality:

Page 3: Route & Trip Planner
"Plan routes, track rides, and save favorites"
Not Built
Action Route & Trip Planner does not exist
There is zero route/trip planning functionality in the codebase. No screens, no API resolvers, no database tables. Advertising a feature on the paywall that doesn't exist is a serious issue — it can lead to App Store rejection, refund requests, and negative reviews. You have two options:

Option A: Remove the Route & Trip Planner page from the paywall carousel and replace it with an existing pro feature (e.g., AI Diagnostics, Learning Library, or PDF Export).
Option B: Build the Route & Trip Planner feature before publishing this paywall.
Note "Secure Bike Vault" oversells
The subtitle says "Store registrations, keys, and photos securely." The app stores bike details and photos, but there is no dedicated registration document storage, key management, or secure vault feature. The Garage tab stores make/model/year/mileage/photo. Consider rewording to: "Track your fleet with photos, specs, and service history" — which accurately describes what's built.

Update the RevenueCat paywall localization table to replace both https://example.com entries with the correct URLs above. This is a blocking issue for app store submission.

Design Visual & UX Critique

What works well

Good Carousel-based feature showcase
The 3-page carousel with feature images is a proven paywall pattern. Each page has a clear title + subtitle + image, creating a compelling value proposition flow. Page indicators help users understand there's more to see.
Good Package comparison layout
Showing Annual vs Monthly side-by-side with the annual option displaying per-month breakdown ($6.67/mo) and a discount badge is a well-established conversion pattern. The "Continue" CTA is prominently placed.
Good Footer structure
Restore Purchases, Terms, and Privacy are all present in the footer — required for App Store compliance. Just need to fix the URLs.

Improvement Opportunities

UX No close/dismiss button visible
Users must be able to dismiss the paywall without purchasing. Ensure there's a clear X or "Skip" button at the top. If it's the onboarding paywall, RevenueCat's default close button may be present but verify it's visible and accessible.
Suggestion Add trust signals
Consider adding "Cancel anytime" text near the purchase button and "No payment now" if using free trials. Your in-app upgrade screen already has these elements ("Cancel anytime", "No charge during trial") but the RevenueCat paywall lacks them. These significantly improve conversion.
Suggestion Feature list instead of carousel
Consider supplementing the carousel with a bullet-point feature list below the packages. Users who don't swipe through all pages will only see one feature. A concise feature list ensures all value props are visible at a glance (e.g., "Unlimited bikes", "AI Diagnostics", "Smart Reminders", "PDF Export").
